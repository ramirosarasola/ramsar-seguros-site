DESKTOP · full chain
Inicio / Aseguradoras / Sancor Seguros / Plan Federal
MOBILE · collapsed
Inicio / … / Plan Federal
Anatomy"Inicio" (text, not icon) · separator · 1–N intermediate links · separator · current page (not a link).
Inicio · text vs iconText "Inicio", not a home icon. Justification: Spanish-AR users scan text faster than icon-as-first-item, and screen-readers parse it without an aria-label workaround. The icon is reserved for in-content "Volver al inicio" buttons.
SeparatorForward slash "/" · color sand-300 · same size as link text · padding space-1 (4 px) on each side. Justification: chevron-right would compete visually with §1.4 dropdown chevrons and §3 card "leer más" affordances; the slash is path-like and unambiguous.
Typography · linksbody-s (14 / 22 · 400) · color sand-600. Hover: marine-700 · underline 1 px offset 3 px.
Typography · currentSame size as links · weight 500 (subtle bump) · color sand-900. More prominent, not less: this is the page label and the most important text in the row.
Gapspace-2 (8 px) between every adjacent element (link · sep · link · sep · current). The separator absorbs the 4 px of padding on each side; visible spacing is uniform.
ContainerWidth: matches the page's content container (container-default). Top padding space-5 (20 px), bottom padding space-3 (12 px). Sits between the global nav and the page hero/H1.
Mobile truncationIf the chain has > 3 items and viewport < md (768 px): collapse all middle items into an ellipsis token … styled as a tappable chip (sand-100 background, radius-xs, 4×2 padding). Tap expands the full chain inline. Always preserves Inicio and the current page.
Long page titleCurrent-page label truncates with ellipsis if the row would otherwise wrap. Full title remains in document <h1> and in the schema.
SEO / SchemaRequired: BreadcrumbList JSON-LD emitted in <head>, with every chain item indexed 1..N. Visual order and schema order must match exactly. rel="nofollow" is forbidden on breadcrumb links — they pass equity. Internal anchor text in the breadcrumb is the canonical short title of the target page, never the long title.
ARIA<nav aria-label="Breadcrumb"> wrapping an <ol>. Each <li> contains either an <a> or the current span. Current span carries aria-current="page". Separators are CSS-generated (::before on each <li> except the first) so SRs do not read "slash slash slash".
KeyboardNative — Tab through the links, Enter to follow.
DoMatch each visible label to its BreadcrumbList name 1:1. Mismatched breadcrumb data is the most common reason rich-results validation fails.
Don'tDon't render separators as text inside <li>. They must be CSS pseudo-elements so screen readers don't recite them.
Figma propsdepth: 2 · 3 · 4 · 5 — truncated: boolean (mobile state) — text overrides per crumb. No color variants; breadcrumb is light-bg only.

Illustration · marine + amber
ContainerOuter: full-bleed background (paper, no rounded corners, no max-width on the color band). Inner: container-default (1200 px max · 64 / 24 px outer padding). The hero's color is full-bleed; the hero's content is contained.
HeightDesktop: min-height: 640 px (content-driven). Targets one viewport on a 1366×768 laptop with global nav above (72 px) — leaves the insurer-strip visible without scroll. Tablet (md): 560 px. Mobile (xs/sm): auto height, content-driven, ~480 px typical.
Vertical paddingDesktop: space-24 (96 px) top · space-20 (80 px) bottom. Mobile: space-16 (64) · space-12 (48).
LayoutOption A — Text 55% / Visual 45% — chosen. Justification: (1) The H1 wraps to a clean 2–3 lines at 55% column width with our Display-XL leading, which is essential for SEO crawlers reading line one and humans reading the promise. (2) A centred text + bg-visual hero (Option B) would force the H1 to compete with the illustration's chroma and reduce ranking-text legibility. (3) The 45% visual slot accommodates a real illustration without imagery feeling decorative — it earns its place by anchoring the page brand visually right where the eye scans next after the H1.
Grid2-column · template 1.22fr 1fr (≈55/45) · column gap space-12 (48 px) · align-items center.
Background · colorSolid bg-page (sand-50, #F8F7F4). No gradient. The page is cream-on-cream; the hero is unified with the rest of the document and the visual + amber accents do the chromatic work.
Background · textureOne decorative element: a 1 px marine-200 @ 28% α hairline rule beneath the trust-row, extending the full container width. Reads as "specification line", not "web decoration". Nothing else — no gradient mesh, no noise, no shapes.
Visual · styleIllustration — chosen over photography. Why: (a) Stock photography of Argentine drivers reads templated; original photoshoots cost five figures and freeze the brand into one demographic. (b) Illustration is recolorable via tokens — when palette shifts, the visual ships with it. (c) Illustration carries the "smart friend" tone without the parasocial weight of a photographed face.
Visual · subject"The garaged car at dusk." A 3/4 front view of a generic compact sedan parked under a soft amber pool of light. Mid-flat illustration style — geometric shapes with restrained gradient shading, no outlines, no faces, no logos. Shown in place of content during async load. Skeletons preserve the layout, prevent jarring height shifts when content arrives, and communicate "loading" without a spinner blocking the page.

  • wide: 75% width · same height

  • narrow: 30% width

Heading skeletonHeight 24 px · width 60% · radius-xs.

Hand-off ruleWhen real content arrives: skeleton fades out (opacity 1 → 0, duration-default · ease-out) while content fades in (opacity 0 → 1, same duration, 80 ms delay). Dimensions must match exactly — any size mismatch causes layout shift, which defeats the purpose.

DoMatch skeleton dimensions to real content exactly. Any size mismatch causes Cumulative Layout Shift — which Google measures and ranks.
Don'tDon't use a spinning loader where a skeleton works. Spinners block the page; skeletons preserve the user's spatial memory of where things will be.
