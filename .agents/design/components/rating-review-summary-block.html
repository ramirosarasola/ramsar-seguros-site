Rating / Review Summary Block

The aggregate-rating block at the top of every insurer page. Big number on the left, distribution-bar histogram on the right. Built so the visitor knows in two seconds whether reviews skew positive, mixed, or hostile.

ContainerNo background, no border — transparent block within an insurer-page section. Width 100% of container-default. Used once per insurer page.
Layout · desktopHorizontal grid · template auto 1px 1fr · column gap space-8 (32 px) · align-items center. Left = aggregate score · 1 px vertical divider sand-200 · right = breakdown bars.
Layout · mobileStacks vertically · score on top centered · vertical rule replaced by horizontal 1 px sand-200 divider with space-5 (20 px) padding above/below · breakdown below.
Score numberTypography display-l serif (56 · 400 · −0.02 em). Note: weight 400, not 700 — Instrument Serif at this scale needs no extra weight; bold reads as institutional rather than authoritative. Color sand-900. font-variant-numeric: tabular-nums. Always one decimal place (4.4, not 4.40 or 4).
"/5" suffixTypography h4 sans (22 · 400) · color sand-400 · baseline-aligned with score · 2 px gap.
Stars row24 px stars · color amber-500 filled · sand-300 empty. Half-stars via CSS clip-path on duplicate star. Letter-spacing +0.06 em for breathing. Top margin from score space-1 (4 px).
Review countFormat: "312 reseñas verificadas". Typography body-s (14 · 400) · color sand-500. The count itself bumps to sand-700 and weight 500 to emphasize the credibility quantum. Top margin space-1 (4 px) from stars.
Breakdown · structure5 rows, top to bottom: "5 estrellas" → "1 estrella". Each row: 3-column grid · template 70px 1fr 40px · column gap space-2 (8 px) · align-items center.
Breakdown · labelTypography label-s mono (11.5 · 500) · color sand-700. Format: "X estrellas" (singular "1 estrella" — accurate Spanish, not lazy localization).
Breakdown · trackHeight 8 px. Border-radius radius-full. Background sand-200.
Breakdown · fillBackground amber-500 (consistent with stars — the warm color carries "review weight" throughout). Border-radius radius-full. Width = (rating bucket count ÷ total count) × 100%.
Breakdown · countTypography label-s mono (11.5 · 400 · tabular-nums) · color sand-500 · text-align right.
Row gapspace-2 (8 px) between rows.
AnimationOn viewport entry (IntersectionObserver, threshold 0.5): each bar's fill animates from 0% → final % over duration-slow (360 ms) · ease-out. Bars stagger 100 ms apart, top-down (5★ first, 1★ last). Score number counts up from 0 → final via requestAnimationFrame ease-out over 600 ms · tabular-nums prevents horizontal jiggle. Reduced motion: bars render at final width immediately, score appears final.
AccessibilityContainer role="group" aria-label="Resumen de reseñas: 4.4 de 5 estrellas basado en 312 reseñas". Each breakdown row carries aria-label="194 reseñas con 5 estrellas" — bars are decorative, the spoken label carries the data. Stars use aria-hidden; the aggregate score number is the meaningful text.
SEO · schemaMust emit AggregateRating JSON-LD attached to the Organization entity for the insurer page: ratingValue · reviewCount · bestRating: 5 · worstRating: 1. Schema and visible UI must agree exactly.
DoAnimate bars and score together on scroll-in. The motion communicates "real data computed live" — the same gesture financial dashboards use to signal liveness.
Don'tDon't pluralize "estrellas" universally. Spanish pluralization is grammatical — "1 estrella" not "1 estrellas". Mixed locale on a UI block this prominent looks careless.
Figma propsscore: 0.0–5.0 step 0.1 — reviewCount: number — bucket[5..1]: percentages — text override for "reseñas verificadas" label.
