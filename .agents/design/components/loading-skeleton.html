§ 5.7

Loading Skeleton

Shown in place of content during async load. Skeletons preserve the layout, prevent jarring height shifts when content arrives, and communicate "loading" without a spinner blocking the page.

SKELETON · blog card · text lines · insurance card
Color tokensBase color sand-200. Shimmer pass-through sand-100. Implementation: background: linear-gradient(90deg, sand-200 0%, sand-100 50%, sand-200 100%); background-size: 200% 100%.
AnimationBackground-position 200% 0 → −200% 0 · duration 1500 ms · timing linear · iteration infinite. Wave moves left → right.
Border radiusMatches the element being represented. Text lines: radius-xs (4 px). Images: radius-md (12 px). Avatars: radius-full. Buttons: radius-sm (8 px).
Text line widths
  • full: 100% width · height 16 px
  • wide: 75% width · same height
  • medium: 50% width
  • narrow: 30% width
Used in sequence to simulate paragraphs (e.g. full, full, wide, medium).
Heading skeletonHeight 24 px · width 60% · radius-xs.
Avatar skeletonCircle at the represented avatar's exact size (24 · 32 · 40 · 64 · 72 px).
Image skeletonAspect-ratio matches its target (e.g. aspect-ratio: 16/9 for blog hero images). Radius matches.
Composite · blog card16:9 image block + 2-line headline (full · 65%) + 3-line excerpt (full · full · 75%) + metadata row (24 px avatar circle + two short lines). Container takes the real blog-card border + radius. Total height matches real card height to prevent CLS.
Composite · insurance card40 px logo block + 18 px name line + 5 × 14 px star circles + 3 bullet lines (full · 90% · 80%) + 16 px price line + 44 px button block. Real card border + radius preserved.
Composite · table rowPer row: 1 left-column label (medium-width) + N right-column cells (each: 16 px square for icon, or 60% width line for numeric value). Row height matches real table row.
Reduced motionShimmer animation paused. Skeletons remain visible as static sand-200 blocks. The waiting state is communicated by presence, not by motion.
Hand-off ruleWhen real content arrives: skeleton fades out (opacity 1 → 0, duration-default · ease-out) while content fades in (opacity 0 → 1, same duration, 80 ms delay). Dimensions must match exactly — any size mismatch causes layout shift, which defeats the purpose.
AccessibilityWrapper carries aria-busy="true" aria-live="polite". Once content arrives, aria-busy drops to false. Skeleton blocks themselves are decorative (aria-hidden).
DoMatch skeleton dimensions to real content exactly. Any size mismatch causes Cumulative Layout Shift — which Google measures and ranks.
Don'tDon't use a spinning loader where a skeleton works. Spinners block the page; skeletons preserve the user's spatial memory of where things will be.
Figma propstype: text-line · heading · avatar · image · button · composite-card · composite-row — width: full · wide · medium · narrow (for text variants) — animated: boolean.
