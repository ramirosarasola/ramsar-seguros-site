§ 5.6

Toast Notification

Transient floating feedback after an action. Same content shape as Alert (icon + title + description) but lives in a fixed-position stack and disappears automatically. Used for "Cotización guardada", "Datos copiados", "Sin conexión".

Position · desktopFixed · top-right · offset space-6 (24 px) from top and right edges of viewport.
Position · mobileFixed · top-center · offset space-4 (16 px) from top edge · horizontal padding space-4 (16 px) from viewport edges.
WidthDesktop fixed 360 px. Mobile calc(100vw − space-8) (full minus 2× 16 px gutters).
ContainerBackground sand-0. Radius radius-md (12 px). Shadow elevation-3. Border-left 3 px in variant color. Padding space-4 (16 px). Layout: flex · gap space-3 (12 px) · align-items center.
StackingMultiple toasts stack vertically, newest on top (visually above previous), gap space-2 (8 px). Maximum 3 visible simultaneously. A 4th toast pushes the oldest off (oldest exits with slide animation).
AnatomyIdentical to §5.5 Alert (icon · title · description) plus an optional inline action button (variant btn-ghost-sm) right of the description (replaces dismiss-X when present). When both action and dismiss are needed, action sits inline, dismiss-X stays top-right.
TypographyTitle: label-m (13 · 600) · color sand-900. Description: body-xs (13 · 400) · color sand-600 — slightly tighter than alerts because the toast is read in a glance.
Auto-dismissSuccess: 5 seconds. Info: 5 seconds. Warning: 8 seconds (warnings need extra read time). Error: never auto-dismiss — errors require user acknowledgement. Toast with an action: never auto-dismiss (the action is the dismissal).
Pause on hoverYes — auto-dismiss timer pauses on pointer-enter, resumes on pointer-leave. Implemented as setTimeout pause/resume, not just visual freeze.
Entry animationFrom translateX(calc(100% + 24px)) + opacity 0 → translateX(0) + opacity 1. Duration duration-default (220 ms) · ease-out. Mobile: translateY(-100%) → 0 (slides from top).
Exit animationReverse · duration duration-fast (120 ms) · ease-in. Faster than entry.
Progress indicatorOptional 2 px bar along the bottom edge of the toast, color = variant-default. Width animates 100% → 0 over the dismiss duration. Provides a visual countdown without intruding. Disabled in reduced-motion.
Z-indexz-90 · 90. Above modals (z-80), below the absolute-top emergency layer (reserved).
AccessibilityContainer role="status" (success/info) or role="alert" (error/warning). aria-live="polite" or "assertive" respectively. aria-atomic="true". Stack container is role="region" aria-label="Notificaciones". Toast focus management: focus does not shift to toasts on appearance (would steal focus from user's current task); action button is reachable via Tab.
DoMake error toasts persistent. An error that auto-dismisses before the user reads it is a worse experience than no error at all.
Don'tDon't use toasts for things the user expected to happen. "Saved" after a deliberate "Save" click is unnecessary noise — only use toast feedback for ambient or asynchronous actions.
Figma propsvariant: success · warning · error · info — showAction: boolean — showProgress: boolean — state: entering · resting · exiting — text overrides for title · description · action.
