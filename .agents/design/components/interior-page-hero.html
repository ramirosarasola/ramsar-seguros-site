§ 2.2

Interior Page Hero

Used on insurer landing pages, vehicle brand pages, and static pages. Shorter, centred, no illustration — its job is to set the page topic clearly so users and crawlers know what they're looking at, then get out of the way.

VARIANT A · Brand tint · insurer page
Inicio/Aseguradoras/Sancor Seguros
Aseguradora destacada

Seguros Sancor en Argentina

Planes, coberturas, opiniones de clientes y comparativa actualizada de Sancor Seguros para autos.

VARIANT B · Neutral · vehicle page

Seguros para Toyota Corolla

Cotizá tu Corolla con 14 aseguradoras del país.

LayoutCentered text, no visual element. Justification: an interior page hero is a wayfinding signal, not a sales surface. Centering the H1 makes the page topic unambiguous in one glance and works equally for "Sancor Seguros" (proper noun, balanced visual weight) and "Seguros para Toyota Corolla" (longer string).
Containercontainer-default · content centred horizontally · inner content max-width 680 px.
HeightDesktop: min-height 280 px · vertical padding space-16 (64 px) top · space-12 (48 px) bottom. Mobile: min-height 200 px · padding space-10 / space-8 (40 / 32 px).
Elements (top → bottom)
  1. Breadcrumb (§1.5) · centered · bottom margin space-4 (16)
  2. Optional category/status badge · bottom margin space-3 (12)
  3. H1 · centered · bottom margin space-3 (12)
  4. Subtitle/description · centered · bottom margin space-6 (24) (or omitted if no CTA follows)
  5. Optional CTA · centered
Badge above H1Variant badge-md (per §5.4). Color variant by context: Aseguradora destacada → marine-50 bg / marine-700 text. Vehículo popular → amber-300 / amber-700. Static page → omit.
H1 · typographyh1 (44 / 50 · −0.018 em · serif · 400). Color depends on variant background (see colors below). Max-width 22 ch for clean 1–2 line wrap.
Subtitle · typographybody-l (18 / 30 · 400) · color text-secondary (sand-700) · max-width 56 ch · max 2 lines (truncate with ellipsis if longer; full copy goes in body).
Optional CTAWhen present: btn-primary-md · single button only. Used on insurer pages ("Cotizar con Sancor"), omitted on most vehicle pages and static pages (vehicle page CTA lives in §4.7 inline-CTA further down).
Variant A · Brand tintFor insurer landing pages. Background marine-50 (#ECF4F3). H1 color marine-700. Subtitle sand-700. Justify: visually signals "this is an insurer surface within our system"; the tint pairs with insurer logos in Column 2 of the footer.
Variant B · NeutralFor vehicle brand/model pages. Background sand-100 (#EFEEE9) — one step warmer than page bg. H1 color sand-900. Subtitle sand-600. Justify: vehicles are the user's territory, not Ramsar's — staying chromatically neutral lets the auto brand stand on its own.
Variant C · White + borderFor static pages (Contacto, FAQ, Nosotros, legal). Background sand-0 (white). 1 px border-subtle (sand-200) on bottom. H1 color sand-900. Subtitle sand-600. Justify: signals "utility page" — no brand campaign, no insurer affiliation; just information.
ResponsiveLayout unchanged across breakpoints (centred, single column). Only vertical padding shrinks (see Height). Breadcrumb truncates on mobile per §1.5.
AccessibilitySingle <h1> per page. Breadcrumb precedes it in DOM. Optional badge is decorative (aria-hidden) — its text is not navigational.
SEOThe H1 is the most semantically important text on the page. On insurer pages, format: "Seguros [Insurer] en Argentina" (matches likely search query). On vehicle pages: "Seguros para [Marca] [Modelo]".
DoMatch the variant to the page type strictly: brand tint = insurer, neutral = vehicle, white = static. The chromatic signal helps users orient across sections.
Don'tDon't add a hero illustration here. The illustration belongs to the homepage; reusing it on every page drains it of meaning.
