§ 5.8

Empty State

What a section looks like when it has nothing to show. Three variants — no results, no content yet, error loading. Each variant carries an illustration concept, copy, and an optional CTA that converts the empty moment into the next action.

EMPTY · NO RESULTS · centered · 400 px max
No encontramos resultados

Intentá con otros términos o explorá todas las aseguradoras.

LayoutVertical stack · centered horizontally · max-width 400 px. Vertical padding space-12 (48 px) top and bottom (relative to the empty container's bounds).
IllustrationSize: 120 × 120 px square. Style: flat SVG · 2 colors maximum — marine-200 + sand-300. No shadows, no gradients · clean geometric primitives only. Sits inside a 16 px marine-50 rounded square (radius-lg) as a visual frame — the frame is the only chromatic surface in the empty state.
HeadlineTypography h5 (18 / 26 · 600 · sans) · color sand-700 (slightly muted from sand-900 — empty states should feel calm, not stark). Top margin from illustration: space-4 (16 px).
DescriptionTypography body-s (14 · 400) · color sand-500. Max 80 characters · 1–2 line wrap. Top margin space-2 (8 px). Text-align center.
CTA (optional)Variant btn-primary-md. Top margin space-6 (24 px). Width: auto (centered). Always optional — many empty states resolve naturally (e.g. "Pronto habrá contenido aquí" needs no action).
Variant · no search results

Illustration concept: A magnifying-glass outline (marine-200 stroke 2 px) with a sand-300 dashed circle inside representing "nothing found". 120 × 120 artboard, centered.

Headline: "No encontramos resultados"

Description: "Intentá con otros términos o explorá todas las aseguradoras."

CTA: "Ver todas las aseguradoras" → navigates to the listing index.

Variant · no articles yet

Illustration concept: A blank document outline (3 thin horizontal lines suggesting page rules, marine-200) overlapped by a pencil at 30° (sand-300). Communicates "in progress" without becoming literal.

Headline: "Pronto habrá contenido aquí"

Description: "Estamos trabajando en nuevos artículos para esta categoría."

CTA: none. Linking back to the main blog index is offered below the empty state in the page footer, not inside it.

Variant · error loading

Illustration concept: A cloud shape (marine-200 outline) with a small lightning-bolt sand-300 stroke cutting diagonally through it. Reads as "interrupted" rather than "broken" — calmer than a sad-face cloud, more diagnostic.

Headline: "Algo salió mal"

Description: "No pudimos cargar el contenido. Intentá de nuevo."

CTA: "Reintentar" — triggers the parent component's load function (not a full-page reload).

ContainerThe empty state inherits the parent section's bounds. No internal background, no border. The cream paper of the page is the canvas; the empty state is its own quietness.
AccessibilityWrapped in <div role="status" aria-live="polite"> so screen readers announce the empty state when it replaces a previously-populated section. Illustration is aria-hidden. Headline is <h3> within the section's hierarchy.
DoLet some empty states stand without a CTA. Sometimes "Pronto habrá contenido aquí" is the entire message — adding a button to it implies the empty state is a failure rather than a moment.
Don'tDon't draw the illustration with shadows, gradients, or 3+ colors. Empty-state illustrations earn their character from restraint; complexity here reads as cope.
