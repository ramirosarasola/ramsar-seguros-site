§ 5.1

Input Field

The cotizador's atomic unit. Used everywhere a user types: DNI, license plate, model year, email, phone, address. Designed to meet a 44 px touch-target floor at every viewport and to communicate state through border + halo, not through size shifts.

DimensionsHeight 44 px (touch-target floor). Padding space-3 (12 px) vertical · space-4 (16 px) horizontal. Radius radius-sm (8 px). Border 1 px border-default (sand-300). Background sand-0.
LabelPosition above input. Typography label-m (13 · 500 · sans) · color sand-800. Bottom margin space-1 (4 px). Always above, never floating. Floating labels measurably increase form-error rates (Baymard); they sacrifice the constant context label-above provides for an aesthetic that disappears the moment a user types.
Required indicator"*" trailing the label · color error-default (#B8341C) · weight 700 · no space before "*". Optional fields receive a trailing "(opcional)" suffix in caption · sand-500 instead — Argentine forms expect explicit optionality more than optional indicators.
Input textTypography body-m (16 · 400 · sans) · color sand-900. Note: 16 px is non-negotiable — iOS Safari zooms the viewport on focus of any input < 16 px, which is the worst possible UX on a cotizador.
PlaceholderTypography body-m (16 · 400) · color sand-400. Used sparingly — only for format hints ("AB 123 CD"). Never used to repeat the label.
Helper textBelow input · top margin space-1 (4 px). Typography body-xs (13 · 400 · +0.005 em) · color sand-500. One line preferred; wraps to two maximum.
Error messageReplaces helper text (does not stack with it). Typography same as helper · color error-dark (#7A1F0F) · weight 500. Leading icon circle-alert · icon-xs · color error-default · gap space-1 (4 px). Copy formula: stating what is wrong + what's required. Example: "El DNI debe tener entre 7 y 8 dígitos" — not "Inválido".
Prefix iconInside input · left edge · 16 px left padding · icon-sm (16 px) · color sand-400 rest, marine-700 on focus. Input text re-indents to 40 px left padding when prefix present.
Suffix iconInside input · right edge · 16 px right padding · icon-sm · same colors. Used for show/hide password, clear-value (x-circle), copy-to-clipboard.
Character countBottom-right of input · typography caption mono (11 · 400 · tabular-nums) · color sand-400 · format "X / Max". Only renders when max-length is set. Switches to error-default when characters > max.
States · defaultAs specified above.
States · hoverBorder sand-300 → sand-400. Transition 120 ms · ease-out. No bg change.
States · focusBorder marine-700. Halo: box-shadow: 0 0 0 3px rgba(15,67,63,0.18). Background stays sand-0. Transition 120 ms · ease-out. Caret color marine-700.
States · filledBorder stays sand-300. Background stays sand-0. Helper text remains; no visual change beyond text presence.
States · errorBorder error-default. Background error-light (#F6E2DD) at 30% α (cream-toned blush, not pink). Focus ring color shifts to error-default @ 18% α. Label color unchanged (errors don't punish the label).
States · disabledBackground sand-100 · border sand-200 · text sand-400 · label opacity 0.5 · cursor: not-allowed. Common pattern: a downstream field disabled until a prerequisite is selected; placeholder copy explains the prerequisite ("Seleccioná primero la marca").
States · read-onlyBackground sand-50 · border sand-200 · text sand-900 (preserved — read-only is still readable data). No focus ring. Cursor default.
AccessibilityLabel uses <label for>; never aria-label only. Error message linked via aria-describedby; aria-invalid="true" in error state. Required fields use aria-required="true" + the "*" indicator — never required attribute alone (browser-default validation messages are inconsistent and untranslated).
DoKeep input text at 16 px minimum on mobile. Anything smaller forces iOS Safari to zoom in on focus, which is the single most jarring UX moment in a 2-minute form.
Don'tDon't show both placeholder and label in error state. Pick one signal — the user needs guidance, not a duplicate of the label.
Figma propsstate: default · hover · focus · filled · error · disabled · read-only — required: boolean — showPrefixIcon · showSuffixIcon · showHelper · showError · showCount: booleans — text overrides for label, placeholder, helper, error, value.
