GridDesktop: 3 equal columns · column gap space-6 (24 px). Mobile: vertical stack, not horizontal scroll. Justification: on a phone, comparison is a secondary task to comprehension — readers don't compare plans by swiping at thumbnail size; they scroll one plan at a time. A horizontal-scroll carousel hides plans the user didn't ask to hide. Stacked: the "recomendado" plan moves to position 1 on mobile (above the others) so the recommendation is the first encounter.
Card · standardWidth: equal in 3-column grid. Padding space-6 (24 px). Background sand-0 · border 1 px sand-200 · radius radius-lg (16 px) · shadow elevation-0.
Card · recomendadoBorder 1.5 px marine-700 (vs 1 px sand-200). Shadow elevation-2 (vs elevation-0). Background remains sand-0 — no tint. Tinting plus border-emphasis is double-treatment; one signal is enough. Scale: transform: scale(1.02) on desktop only. Justify: a 2% lift breaks the row's visual rhythm just enough for the eye to settle on the middle card, without making the side cards feel "demoted". On mobile the scale is disabled (every plan is full-width, scale is invisible).
"Más elegido" badgePosition absolute · top: −12 px · left: 50% · transform: translateX(−50%) (floats above card, horizontally centered). Background amber-500 · text midnight-700. Padding 4 px / 14 px. Radius radius-full. Typography label-s (11.5 · 700 · +0.02 em) — slightly heavier (700) than the §3.1 popular badge because this is a more prescriptive recommendation.

  5. 1 px divider after price. Bottom margin space-4 (16)
  6. Coverage list (6–8 items)

Coverage list

Included item: check icon (icon-sm · stroke 1.75 · color success-default (#1F7A4D)) + label. Label typography body-s (14 · 400) · color sand-800. Icon → label gap space-2 (8 px).

Excluded item: x icon (icon-sm · color sand-300) + label. Label typography same size · color sand-400. Justification: excluded items visually recede (light grey on light grey) — they remain readable but communicate absence without shouting. This is a deliberate departure from the marine-700 check used on §3.1: in plan comparisons, the user is reading differences; success-green vs sand-300 makes the contrast scannable in a single sweep.

Item gap: space-3 (12 px) between items.

Count: 6–8 items per card. Order is identical across all three cards — same row in card 1 must describe the same coverage in card 2 and card 3, so the eye can scan horizontally across cards on desktop.

CTA button · standard cardsVariant btn-outline-md · full card width · top margin space-6 (24 px). Label "Cotizar este plan".
CTA button · recomendado cardVariant btn-primary-md · full card width · same label. Visual contrast (solid vs outline) is the second differentiator after the border treatment.
DoAlign coverage-list items row-for-row across the three cards. Visual alignment is what makes the comparison real.
Don'tDon't tint the recommended card's background. The border + shadow + scale + CTA inversion are already the recommendation. Adding fill is shouting.
Figma propsvariant: standard · recomendado — itemCount: 6 · 7 · 8 — showDescription: boolean — text overrides for plan name, price, description, all coverage items (boolean per item: included · excluded).
