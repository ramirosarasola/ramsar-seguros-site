§ 3.8

Feature / Benefit Block

The system's least-decorated card. Used in 3- or 4-column feature grids ("Por qué Ramsar", "Qué incluye tu cobertura"). Informational, not interactive — no hover state, no link.

VARIANT A · light bg · iconbox
14 aseguradoras
Comparamos en simultáneo las pólizas más elegidas del país sin sesgos comerciales.
VARIANT B · dark bg · icon-only
Sin compromiso
Cotizá sin tarjeta, sin firma, sin trampas. Sin nada que después aparezca.
VariantsA · On light background (cream/white sections): icon sits inside a colored container. B · On colored background (marine-700 sections, midnight footer): icon sits naked, container removed, stroke recolored. The choice is contextual to the surrounding section, not a Figma boolean of the component itself.
Variant A · icon containerShape rounded square (radius-md (12 px)) · size 48 × 48 px. Background marine-50. Icon inside: icon-md (20 px) · stroke 1.75 · color marine-700. Centered both axes.
Variant B · iconNo container. Icon icon-lg (24 px) · stroke 2 · color sand-0 (or amber-500 for emphasis-block use). Sits flush-left of the title with no padding.
TitleTypography h5 (18 / 26 · 600 · sans). Color: sand-900 on light · sand-0 on dark. Top margin from icon: space-3 (12 px). Max 1 line on desktop (24 ch); wraps to 2 lines on mobile.
DescriptionTypography body-s (14 / 22 · 400). Color: sand-600 on light · sand-200 @ 78% α on dark. Max 120 characters (≈ 3 lines at column width). Top margin space-2 (8 px).
Block dimensionsNo fixed width — fills the column. No background, no border, no shadow (the block is the content). Vertical internal alignment: top-aligned within the column so multi-row grids align on the icon row.
Grid layoutDesktop: 3 or 4 columns (section-dependent; 3 for emphasis blocks · 4 for compact feature grids) · gap space-8 (32 px) horizontal · space-12 (48 px) vertical (when wrapping to a second row). Tablet: 2 columns. Mobile: 1 column · gap space-8 (32 px).
StatesNone. This is an informational block. No hover, no focus, no link wrapping. If it needs a CTA, the section that contains it gets one — not the block itself.
AccessibilityTitle is <h3> within the surrounding section (per page structure). Icon is aria-hidden. The block is plain content — no role override needed.
DoTreat the feature block as text-first. The icon decorates the heading; the heading decorates the description; the description is the work.
Don'tDon't add a hover lift "for consistency with cards". Feature blocks are paragraphs with icons, not interactive surfaces.
Figma propsvariant: light · dark — icon: instance-swap — text overrides for title and description.
