Coverage Comparison Table

The system's most complex content surface. Lives on insurer landing pages and on vehicle-model pages. Built so the eye can scan plans horizontally and features vertically without losing position.

OrientationRows = coverage features · Columns = plan tiers. The user's question is "which plan covers what I care about?" — they scan a feature row to compare plans on that feature, then drop to the next feature. Inverting to columns-as-features would force horizontal scanning across longer cells and ruin the comparison rhythm at 320 px.
ContainerWidth 100% of container-default. Min-width 600 px — below this the table enters horizontal-scroll mode. Border 1 px sand-200 · radius radius-lg (16 px) · overflow: hidden.
Header row · height & backgroundHeight 64 px. Background marine-700 for standard columns · marine-800 for the recommended column (one step darker). Border-bottom 1 px marine-900.
Header · feature column (top-left cell)Background matches the header row (marine-700). Typography label-m mono (13 · 500) · color amber-300 · padding space-4 / space-5 (16 / 20 px) · text-align left. Label: "Cobertura".
Header · plan columnsTypography label-l (15 · 600 · sans) · color sand-0 · text-align center. Recommended column carries a leading ★ (filled star · color amber-500) before the plan name with 6 px gap. "Recomendado" eyebrow is rendered as a tiny caption mono (11 · +0.12 em · uppercase · amber-300) above the plan name, line-height 1.
Header · sticky behaviorposition: sticky · top: 72 px (matches sticky nav height). On scroll past the table top, the header detaches from the table border and gains elevation-2 shadow on its bottom edge. Border-radius corners (top-left + top-right) flatten to 0 once sticky.
Left column · width40% of table on desktop · 52% at md · min 200 px on mobile horizontal-scroll.
Left column · backgroundsand-50 (sticks visually with the page background, distinguishes feature labels from data cells).
Left column · stickyposition: sticky · left: 0. On horizontal scroll, gains a 1 px sand-300 right border + elevation-1 shadow on its right edge.
Left column · feature labelTypography body-s (14 / 22 · 400) · color sand-800 · text-align left · padding space-3 / space-5 (12 / 20).
Left column · tooltip iconOptional info icon (icon-xs · color sand-400) trailing the feature label · 6 px gap. Click/hover opens a §5.x tooltip (max-width 280 px). Hover color marine-700. Used on technical features like "Franquicia", "Cobertura MERCOSUR".
Data cells · height52 px. Padding space-3 / space-4 (12 / 16). Vertical-align middle · text-align center.
Data cells · includedcheck icon · 20 px · color success-default (#1F7A4D). No text accompanies the check unless a coverage limit is specified.
Data cells · excludedminus icon (Lucide · horizontal line) · 20 px · color sand-300. Not "x" — minus reads as "not applicable", x reads as "denied". The semantic distinction matters for plan comparison.
Data cells · numeric valuesTypography body-s mono (14 · 500 · tabular-nums) · color sand-900. Format: "$15.000" (Argentine thousands separator).
Data cells · text valuesTypography body-s (14 · 500) · color sand-900. Used for qualitative differences ("Nacional" · "MERCOSUR" · "Sin tope").
Row stripingOdd rows sand-0 · even rows sand-50. Border-bottom 1 px sand-200 between rows (no border on the last row before the table footer). Both stripes survive in the recommended column with a 4% Marine tint mixed in.
Row hoverEntire row background → marine-50 over 120 ms · ease-out. Survives across the recommended column (the marine-50 wins over the marine-tint). Hover applies on desktop only.
Recommended column · highlightData cells use background marine-50 (slightly stronger than the standard zebra). Left edge 1.5 px marine-700 · right edge 1.5 px marine-700. The vertical strip ties the recommended column to its plan-card sibling (§3.5).
Mobile · horizontal scrollBelow md (768 px) the table scrolls horizontally inside its container. Left column stays sticky (per spec). Native browser scrollbar is hidden via CSS, replaced by a custom 4 px sand-300 thumb on a 6 px sand-100 track at the bottom of the container — always visible to communicate scrollability. Right-edge fade: 24 px linear gradient mask (cream → transparent) hints at off-screen content.
"Ver tabla completa" expandBy default show 8 feature rows. Excess rows are hidden behind a "Ver tabla completa (12 ítems más)" button: variant btn-ghost-md · centered below the table · top margin space-4 (16). Expand: grid-template-rows 0fr → 1fr on the hidden rows wrapper · duration-slow (360 ms) · ease-in-out. Collapse reverses, button relabels to "Mostrar menos".
AccessibilityReal <table> with <caption class="sr-only">Comparación de coberturas</caption>. Header cells use <th scope="col"> for plan columns, scope="row" on the leftmost cell of each row. Included/excluded icons are aria-hidden with a sibling <span class="sr-only">Incluido</span> or "No incluido". aria-live="off" on the expand region.
SEOThe table is HTML not rendered to canvas/SVG — every feature name is crawlable text. Coverage names match the H3 anchors inside the article body so internal-link queries route to deeper content.
DoUse minus for excluded coverage, not x. "Not included" is a feature of a plan, not a failure of one.
Don'tDon't add a "sticky right-edge fade" desktop too. The fade is a mobile affordance for the scroll affordance — adding it on desktop tells users the table is incomplete.
