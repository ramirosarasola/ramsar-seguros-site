Badge anatomyIcon left · label right. Icon size icon-sm (16 px) · stroke 1.75. Label typography label-s (11.5 · 500 · +0.01 em · sans). Icon → label gap space-2 (8 px). No background, no border, no padding around the badge itself.
Row layout · desktopHorizontal · justify-content center · gap space-6 (24 px). Between every adjacent pair of badges: a 1 px × 16 px vertical rule in sand-200, vertically centered with the badge label baseline.
Row layout · tablet (md)Horizontal · 5 badges wrap to two rows if container width < 720 px (3 + 2 typical). Vertical rules removed when row wraps; replaced by row-gap space-3 (12 px).
Row layout · mobile (xs/sm)Two-column grid (3+2). Rules removed; column-gap space-4 (16 px), row-gap space-3 (12 px). Badge labels can wrap to 2 lines if needed.
Context variant · on cream / whiteIcon color marine-700. Label color sand-700. Vertical rules sand-200. Used in homepage hero footer, cotizador hero, mid-page banners.
Context variant · on marine-700Icon color amber-500. Label color sand-0 @ 88% α. Vertical rules marine-500. Used in dark CTA bands.
Context variant · on midnight footerIcon color amber-500. Label color sand-200 @ 78% α. Vertical rules removed (footer trust strip uses larger gaps instead — see §1.6).
AccessibilityThe row is a <ul role="list">; each badge is an <li>. Icons are aria-hidden; label text carries the meaning. Vertical rules are CSS pseudo-elements (not in DOM) so screen readers don't recite them.
DoKeep the badge set to 5 maximum, in the order listed. More badges read as desperation; reorder breaks the rhythm of reassurance from "safe" → "credible" → "free" → "social-proven" → "fast".
Don'tDon't wrap the row in a card with padding and shadow. The trust row's restraint is what makes it credible; chrome makes it feel marketed.
