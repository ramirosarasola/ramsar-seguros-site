§ 4.2

FAQ Accordion

Eight components in this library would be optional if the SEO didn't depend on it. This one isn't. FAQPage schema + featured-snippet capture make this the single highest-traffic UI surface in long-tail keyword space.

ContainerMax-width container-narrow (720 px) · centered on the page (FAQ is reading-mode content). Vertical padding space-12 (48 px) top + bottom.
Item · structureTrigger row + collapsible answer panel. Both share the same horizontal padding (space-5 / 20 px). Border-bottom 1 px sand-200 on every item (last item retains its border to close the list).
Trigger · closedMin-height 56 px. Padding space-4 / space-5 (16 / 20 px). Background sand-0. Display flex / justify-between / align-center. Cursor pointer.
Trigger · question textTypography label-l (15 / 20 · 500 · sans) · color sand-900. Text-align left. Real question marks at start and end ("¿…?" — Spanish punctuation). Length recommendation 60–80 chars for snippet eligibility.
Trigger · expand iconchevron-down · icon-sm (16 px · stroke 1.75) · color sand-500. Right-aligned via flex.
Trigger · open stateBackground marine-50. Question text color marine-700. Chevron color marine-700 + transform: rotate(180deg). Border-bottom updates to 1 px marine-200. All transitions duration-fast (120 ms) · ease-in-out.
Answer panelPadding space-2 (8) top · space-5 (20) right · space-5 (20) bottom · space-5 (20) left. Background marine-50 (continuous with open trigger — no internal divider). Border-bottom 1 px sand-200 (reverts to standard).
Answer · typographybody-m (16 / 26 · 400) · color sand-800. Max-width 62 ch for readability.
Answer · allowed contentPlain text, links (color marine-700 · underline), short bulleted lists (3–6 items max), one inline CTA link at the end. No images, no tables, no nested accordions — those belong in the article body, not in a snippet-eligible block.
Expand animationgrid-template-rows: 0fr → 1fr on the panel's wrapper · duration-default (220 ms) · ease-out. The body content itself does not fade — the height reveal is the entire effect. Chevron rotation runs synchronously.
Collapse animationReverse · duration-fast (120 ms) · ease-in. Faster than open by design.
Single-open ruleOne item open at a time. Opening item B closes item A automatically. Justification: insurance FAQ entries are dense; allowing multiple opens turns the page into a wall of paragraphs the user didn't ask for. Single-open keeps each answer a deliberate read.
KeyboardEnter / Space: toggle. ↓ / ↑: focus next / previous trigger. Home / End: focus first / last. Esc: close currently-open item, focus returns to its trigger. Tab: leaves the accordion entirely (does not iterate through panel contents on open — preserves linear flow).
ARIATrigger is a <button> with aria-expanded, aria-controls="answer-{id}". Answer panel role="region" id="answer-{id}" aria-labelledby="trigger-{id}". Hidden state uses hidden attribute, not CSS display alone.
SEO · schemaRequired: FAQPage JSON-LD embedded in <head>. Every visible question/answer pair must appear in schema 1:1. Schema text and visible text must match character-for-character — Google penalizes mismatches as cloaking. Targets featured-snippet capture for "¿cómo cotizo un seguro de auto?" and similar.
DoPhrase questions exactly as a Spanish-AR user would type them. Voseo, contractions, real punctuation — match the query, capture the snippet.
Don'tDon't allow multiple items open simultaneously. The "all-open" affordance feels generous and reads as overwhelming.
Figma propsstate: closed · open — itemCount: 3–12 — showCtaLink: boolean per item — text overrides for question + answer.
